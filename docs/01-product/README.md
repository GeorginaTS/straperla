# 📦 Product

Aquesta carpeta recull tota la documentació relacionada amb la definició del producte.

L'objectiu és entendre el problema, definir la proposta de valor i validar les decisions abans d'iniciar el desenvolupament.

## Documents

| Document               | Descripció                                        |
| ---------------------- | ------------------------------------------------- |
| `PRODUCT-BOOK.md`      | Visió general del producte.                       |
| `PROBLEM.md`           | Problema que resol Straperla.                     |
| `TARGET-AUDIENCE.md`   | Públic objectiu i segments d'usuaris.             |
| `VALUE-PROPOSITION.md` | Proposta de valor.                                |
| `USER-PERSONAS.md`     | Perfils representatius dels usuaris.              |
| `USER-STORIES.md`      | Necessitats expressades com a històries d'usuari. |
| `MVP.md`               | Definició de la primera versió del producte.      |

---

## Objectiu de la fase

Abans d'escriure una línia de codi, hem de poder respondre amb claredat aquestes preguntes:

* Quin problema volem resoldre?
* Per a qui el resolem?
* Per què la nostra solució aporta valor?
* Quina és la mínima versió del producte que podem validar?
