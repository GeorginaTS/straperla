# 🎨 Design System

Aquest document defineix els criteris que permeten construir una interfície coherent i fàcil de mantenir.

L'objectiu no és definir components concrets, sinó establir les normes que guiaran la seva creació i evolució.

---

## Principis

### Consistència

Un mateix problema s'ha de resoldre sempre de la mateixa manera.

La coherència facilita l'aprenentatge i genera confiança.

---

### Reutilització

Abans de crear un component nou, comprovem si un component existent ja resol el problema.

Només es creen nous components quan és realment necessari.

---

### Simplicitat

Cada component ha de tenir una única responsabilitat.

Quan un component es torna massa complex, cal simplificar-lo.

---

### Evolució

El sistema evoluciona a mesura que evoluciona el producte.

Les noves necessitats fan créixer el sistema; les preferències personals, no.

---

## Elements del sistema

El Design System està format pels documents següents:

* `COLORS.md`
* `TYPOGRAPHY.md`
* `GRID-SPACING.md`
* `ICONOGRAPHY.md`
* `COMPONENTS.md`
* `SCREEN-PATTERNS.md`

Cada document defineix una part del sistema visual.

Junts formen una única font de veritat.

---

## Abans de crear un component

Abans d'afegir un component nou ens fem aquestes preguntes:

* Ja existeix un component que resol aquest problema?
* Es pot reutilitzar sense modificar-lo?
* Manté la coherència del sistema?
* Fa la interfície més simple?
* Aporta valor real a l'experiència?

Si la resposta és "sí", probablement també és una bona decisió de disseny.

---

> *Un bon sistema fa fàcils les bones decisions.*
